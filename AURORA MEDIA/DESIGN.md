# DESIGN.md — 株式会社イースト・デイリー コーポレートサイト

`https://www.east-daily.jp/` トップページのデザイン仕様書。
サイト全体のビジュアル言語、情報設計、構造ルールを再構築・派生制作のために言語化したもの。

## 2. ブランドトーン & コンセプト

- **信頼感と実直さ**を打ち出す、広告代理店としての堅実な表現。
- 「広告は結果検証が最重要、効果の最大化を目指し、検証に力を入れています。」というキーメッセージを起点に、**成果志向**を中心コンセプトとして据える。
- 派手すぎない洗練、北海道発の地域密着感、観光・インバウンドに明るいカラーを両立。
- 写真は人・街・仕事の現場を中心に据え、生活者視点のリアリティを重視。

---

## 3. カラーパレット

ロゴが白抜きで運用されていること、ダークトーンのヘッダー/フッターと白基調の本文コントラストが前提。下記は視認レベルから推定した運用値。

### 3.1 基本色

| 役割 | 用途 | 推定値 |
| --- | --- | --- |
| Primary Dark | ヘッダー、フッター、ナビ背景 | `#1A1A1A` 〜 `#222222` |
| Base White | 本文背景、カードベース | `#FFFFFF` |
| Base Gray (Light) | セクション区切り、補助背景 | `#F5F5F5` |
| Text Primary | 見出し・本文 | `#222222` |
| Text Secondary | 日付・キャプション | `#666666` |
| Border / Divider | 罫線、カード境界 | `#E5E5E5` |

### 3.2 アクセント

- 英字の大見出し（`OUR SOLUTION` / `PROMOTION CASE STUDY` など）では **セリフ体の斜体＋ダーク文字** を基調にしつつ、番号やキーワード部分に差し色が入る構成。
- カテゴリチップや「READ MORE」リンクは**シンプルな黒・ダークグレーのテキストリンク**が中心で、過度な原色は用いない。

## 4. タイポグラフィ

### 4.1 基本方針

- 日本語：ゴシック系（`游ゴシック` / `Noto Sans JP` / `ヒラギノ角ゴ` 相当）
- 欧文：セクション見出しに**セリフ体のイタリック**を採用（例：`*OUR* SOLUTION`、`*PROMOTION* CASE STUDY`、`*INFOR*MATION`、`*DAC*-GROUP`、`*REC*RUIT`）。装飾性と格調をもたらす中核的な表現。

### 4.2 階層

| レベル | 例 | スタイル |
| --- | --- | --- |
| Hero Copy | 「広告は結果検証が最重要、効果の最大化を目指し、検証に力を入れています。」 | 大ウェイト日本語ゴシック、改行でリズムを作る |
| Section Label（英） | `OUR SOLUTION` | セリフ斜体・大サイズ・一部強調 |
| Section Label（和） | 「事業案内」 | ゴシック・中サイズ・英語と併記 |
| Card Title | 「#01 観光客を集客したい！」 | 太ゴシック、番号プレフィックス |
| Card Subtitle | 「国内観光メディア / DOMESTIC TOURISM MEDIA」 | 和英併記、英字は小さめのトラッキング広め |
| Body | カード説明文 | 通常ウェイト、13〜15px 相当 |
| Meta | 日付＋カテゴリ（例：`2025.6.17 | プロモーション事例`） | 細字、グレー |

### 4.3 和英併記パターン

すべてのメインセクションで **「英語タイトル＋日本語補足」** の併記を採用。

```
*OUR* SOLUTION    事業案内
*PROMOTION* CASE STUDY   プロモーション事例
MARKET NEWS   マーケットニュース
*INFOR*MATION   お知らせ
ESD TOPICS   トピックス
*DAC*-GROUP   DACグループについて
*REC*RUIT   採用 / スタッフ紹介
```

派生制作時は必ずこのペアリングを踏襲すること。

## 5. レイアウト / グリッド

### 5.1 全体構造

```
┌─────────────────────────────────────┐
│ Global Header（ダーク背景・白ロゴ）      │
├─────────────────────────────────────┤
│ Hero / Main Visual（スライダー4点）     │
│  └ キーコピー                         │
├─────────────────────────────────────┤
│ OUR SOLUTION（事業6分類カードグリッド）  │
├─────────────────────────────────────┤
│ PROMOTION CASE STUDY（実績4件）        │
├─────────────────────────────────────┤
│ MARKET NEWS（ニュース4件）             │
├─────────────────────────────────────┤
│ INFORMATION（お知らせ4件）             │
├─────────────────────────────────────┤
│ ESD TOPICS（社内トピック4件）           │
├─────────────────────────────────────┤
│ DAC-GROUP（親会社/notoリンク）          │
├─────────────────────────────────────┤
│ RECRUIT（採用 + スタッフ紹介）          │
├─────────────────────────────────────┤
│ COMPANY / CONTACT（拠点3箇所）         │
├─────────────────────────────────────┤
│ Global Footer                        │
└─────────────────────────────────────┘
```

### 5.2 グリッド

- **事業案内**：3カラム × 2行 = 6枚のカード（#01〜#06）。PCで3列、タブレットで2列、スマホで1列の標準レスポンシブ。
- **ニュース各種**：4カラム × 1行のサムネイル＋日付＋タイトル構成。各セクション末尾に `VIEW ALL` リンク。
- **会社情報**：3拠点（札幌・函館・東北）を横並びで並列表示、各拠点に Google Map リンク。

### 5.3 余白

- セクション間の上下パディングは広め。ひとつのブロックを独立して読ませる設計。
- カード内部は画像＋メタ＋タイトルの 3ブロック構成で統一。

## 6. コンポーネント仕様

### 6.1 グローバルナビ

- 位置：最上部固定またはスクロール追従（推定）
- 背景：ダーク（`#1A1A1A` 系）
- ロゴ：白抜き（`/ver2020/img/logo_w.png`）
- メニュー項目：
  1. トップ
  2. 事業内容（6サブカテゴリ）
  3. ニュース（4サブカテゴリ）
  4. 採用/スタッフ紹介（2サブカテゴリ）
  5. 会社概要・代表挨拶
  6. お問い合わせ
- `SCROLL DOWN` / `PAGE TOP` 補助ナビあり。

### 6.2 ヒーロースライダー

- 画像4枚ループ（`top_mv01.jpg` 〜 `top_mv04.jpg`）。
- 自動切替を想定（スライダー実装）。
- オーバーレイコピーとして日本語のキーメッセージを重ねる。

### 6.3 サービスカード（OUR SOLUTION）

各カードのフォーマットは厳密に統一：

```
#[番号] [キャッチ（〜たい！ 形式の欲求ベース）]
[事業名（日本語）][事業名（英語 ALLCAPS）]
[事業説明 2〜3行]
READ MORE →
```

番号ラベルは `#01` 〜 `#06` の連番。見出しは**顧客の課題・欲求（〜したい！）**で書き起こすナラティブが特徴。

### 6.4 ニュースカード

- 正方形サムネイル画像（500×500 推定）
- `YYYY.M.D | カテゴリ名` のメタ行
- カードタイトル
- タイトル行クリックで詳細へ遷移

### 6.5 ロゴ & 外部リンクブロック（DAC-GROUP）

- DACグループ本体サイト / note の2ブロック横並び
- イメージ＋キャプション形式

### 6.6 フッター

- COMPANY（3拠点の住所＋Google Map）
- CONTACT（TEL / FAX / お問い合わせフォームリンク）
- Privacy Policy / Brand Safety Policy
- 著作権表記：`© East Daily Co., Ltd. All Rights Reserved.`

## 7. 情報設計（IA）

### 7.1 事業の6分類（固定）

| # | キャッチ | 事業カテゴリ | 英語表記 |
| --- | --- | --- | --- |
| 01 | 観光客を集客したい！ | 国内観光メディア | DOMESTIC TOURISM MEDIA |
| 02 | 低コスト、高リターンの効果を実現したい！ | WEB・スマートフォンメディア | INTERNET MEDIA |
| 03 | 外国人観光客を集客したい！ | インバウンドメディア | INBOUND MEDIA |
| 04 | 適切な方法で求人募集をしたい！ | 求人・リクルートメディア | RECRUITMENT MEDIA |
| 05 | 成果にこだわった自治体プロモーションを実施したい！ | 自治体プロモーション事例 | PROMOTION CASE STUDY |
| 06 | 企画立案から実施までトータルサポート！ | 総合広告 | GENERAL ADVERTISING |

この6分類は**サイトの背骨**。追加・削除時は全体のバランスを見直すこと。

### 7.2 ニュース3系統

1. **プロモーション事例** — 実績ケーススタディ
2. **マーケットニュース** — 業界動向、インバウンド統計、ふるさと納税など
3. **ESD TOPICS** — 採用イベント、社内トピック
4. **お知らせ** — 刊行物発行、人事、受賞など

それぞれ独立したアーカイブページを持ち、トップでは各4件を表示。

## 8. 画像 / ビジュアル指針

- ヒーロー：業務・現場・街の情景を切り取った写真（4カット）。
- ニュースサムネイル：500×500 の正方形が基本。告知バナー調（文字入り）と写真調が混在。
- アイキャッチに文字情報を載せる場合は、ブランドカラーのダーク／ライトに合わせる。
- 「顔が見える」実績表現：キャラクター、イベントブース、ポスター、スタッフ写真など現場感を重視。

---

## 9. ライティング規則

- **語尾**：「です・ます」調で統一。
- **キャッチ**：事業案内は「〜したい！」という**顧客の内的独白**。
- **数字の使い方**：成果を語る際は具体値で（例：「移住相談件数昨対比128%UP！」「訪日外客数は338万人」）。
- **見出しの強調**：英語タイトルの一部を斜体／色変えで強調する表現（例：`*OUR* SOLUTION`）を全セクションで踏襲。
- **和英併記**：セクション見出し・事業カテゴリは必ず和英併記。

---

## 10. アクセシビリティ / レスポンシブ

- ヘッダーのダーク背景＋白文字は十分なコントラスト比を確保。
- カードグリッドは PC 3〜4列 / タブレット 2列 / スマホ 1列の標準ブレークポイント運用を推定。
- 画像には必ず `alt` を付与（現状は主要サムネイルに alt テキストが入っている）。
- `SCROLL DOWN` / `PAGE TOP` のアンカー導線で長尺ページのナビゲーションを補助。

---

## 11. 実装メモ

- 静的 HTML ベース（`/ver2020/` 以下の CSS/画像資産）。
- ニュースは `/news/`, `/promotion/`, `/market_news/`, `/esd_topics/` の4系統で MT または独自 CMS で管理されている可能性が高い（サムネイルパスに `assets_c/YYYY/MM/` 形式があるため）。
- 外部リンク：DACグループ本体、note、Google Map、採用サイト（`/recruit/`）、デジタル事業サイト（`esddigital.jp`）。

## 12. 派生制作時のチェックリスト

- [ ] 英語＋日本語の二段見出しを踏襲しているか
- [ ] 事業6カテゴリの構造を崩していないか
- [ ] ヒーローにキーコピー「結果検証／効果最大化」系の文言があるか
- [ ] ニュースは 4件 + VIEW ALL パターンを守っているか
- [ ] 3拠点（札幌／函館／東北）すべてを正しく記載しているか
- [ ] フッターに DAC グループ所属が明示されているか
- [ ] 画像サムネイルは正方形基調か
- [ ] セリフ斜体の欧文見出しが各セクションに存在するか
